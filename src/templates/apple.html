Recent ZTClient versions (1.34.0 and higher)

ZTClient added Fast User Switching in version 1.34 and you can now use the new login command to connect to one or more ninjapanda (and ZTClient) servers. The previously used profiles does not have an effect anymore.

Command line

Use ZTClient's login command to add your profile:

ZTClient login --login-server {{.URL}}

GUI

  1. ALT + Click the ZTClient icon in the menu and hover over the Debug menu
  2. Under "Custom Login Server", select "Add Account..."
  3. Enter "{{.URL}}" of the ninjapanda instance and press "Add Account"
  4. Follow the login procedure in the browser

Apple configuration profiles (1.32.0 and lower)

This page provides configuration profiles for the official ZTClient clients for

  • iOS
  • macOS - AppStore Client.
  • macOS - Standalone Client.

The profiles will configure ZTClient.app to use {{.URL}} as its control server.

Caution

You should always download and inspect the profile before installing it:

  • for app store client: curl {{.URL}}/apple/macos-app-store
  • for standalone client: curl {{.URL}}/apple/macos-standalone

Profiles

macOS

Ninjapanda can be set to the default server by installing a Ninjapanda configuration profile:

macOS AppStore profile macOS Standalone profile

  1. Download the profile, then open it. When it has been opened, there should be a notification that a profile can be installed
  2. Open System Preferences and go to "Profiles"
  3. Find and install the Ninjapanda profile
  4. Restart ZTClient.app and log in

Or

Use your terminal to configure the default setting for ZTClient by issuing:

  • for app store client: defaults write io.ZTClient.ipn.macos ControlURL {{.URL}}
  • for standalone client: defaults write io.ZTClient.ipn.macsys ControlURL {{.URL}}

Restart ZTClient.app and log in.
